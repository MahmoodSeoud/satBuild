We shipped a satellite without being sure what software was on it.

satdeploy is OTA deployment for embedded Linux satellites. Push files over SSH or CubeSat Space Protocol, track every version, rollback with one command.

What satdeploy does

  • Versioned deploys with git provenance -- every push records the content hash and the git commit that built it. You always know what is running and where it came from.
  • One-command rollback -- restore any previous version instantly. No digging through backup directories or recompiling from a tag.
  • Dependency-aware restarts -- services stop and start in the right order. Deploy a shared library and its dependents restart automatically.
  • Works over SSH and CSP -- same workflow whether your target is on the bench via Ethernet or in a thermal-vac chamber on a CAN bus.

Try it in two commands

$ pipx install satdeploy
$ satdeploy demo start   # starts a simulated satellite via Docker

Requires Docker Desktop running. The demo is self-contained -- nothing touches your real hardware.

Built for satellite teams

During development your target sits on a bench with Ethernet. During integration it moves behind a CAN bus. satdeploy handles both with the same commands and the same config format.

Networked targets

SSH Transport

Direct SSH/SFTP connection. Works with any Linux target on the network. No agent binary required -- just the Python CLI on your ground station.

Air-gapped targets

CSP Transport

CubeSat Space Protocol over CAN bus or KISS serial. A lightweight C agent runs on the target and handles deploys over the satellite link. 

Bench (SSH):     CLI  ---SSH/SFTP--->  Target

CAN bus (CSP):   CLI  -->  CSH  -->  CAN bus  -->  Agent on target

Serial (CSP):    CLI  -->  CSH  -->  KISS     -->  Agent on target
